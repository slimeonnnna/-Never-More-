# 战利品系统设计文档 (Loot System Design)

战利品系统是玩家获取装备和资源的主要方式，分为怪物掉落、地图宝箱和一种特殊的地图宝藏。

## 1. 角色掉落 (Character Drops)

所有角色（包括普通怪物和友军）的掉落物都在其生成时预先决定，并存放在其背包 (`inventory`) 和装备栏 (`equipped`) 中，玩家可以通过“查看”功能提前预览。击败角色后的掉落遵循以下规则：

1.  **背包全掉落**: 玩家**必定获得**该角色背包 (`inventory`) 中的**所有物品**。
2.  **装备掉落（几率性）**: 玩家有**几率额外获得**该角色当前**装备着**的一件物品。
    - **掉落几率**: 这个几率与怪物的阶级挂钩，数值上等同于该阶级的“基础掉落率 (`baseChance`)”。

| 怪物阶级 (Rank) | 装备掉落率 (Base Chance) |
| :-------------- | :----------------------- |
| 喽啰 (Minion)   | 5%                       |
| 精英 (Elite)    | 30%                      |
| 领主 (Lord)     | 30%                      |
| 君王一阶        | 60%                      |
| 君王二阶        | 65%                      |
| 君王三阶        | 80%                      |
| 远古 (Ancient)  | 90%                      |

- **掉落判定**:
    - **成功**: 如果判定成功，系统会从该角色所有**已装备的物品**中随机抽取一件，作为额外战利品给予玩家。
    - **失败**: 如果判定失败，日志中会显示一条提示信息：`“[角色名称]”穿着的装备随着战斗被损坏了。`

- **粘性怪物特殊掉落**: 击败一个带有“粘性”状态的怪物时，其装备掉落率**依然遵循上表**，但其背包中预生成的物品会更多。

## 2. 角色背包生成 (Character Inventory Generation)

角色背包中的物品通过一个基于概率链的机制 (`generateLoot`) 生成，此过程**不再保证至少掉落一件**。

- **生成流程**:
    1.  **基础掉落判定**: 首先，根据怪物阶级对应的 `baseChance` 进行一次判定。如果成功，则至少生成一件装备。
    2.  **连续掉落判定**: 如果第一件装备成功生成，则会立即根据该怪物阶级对应的 `chainChance` 进行第二次判定。此过程 (`while (Math.random() < dropRateConfig.chainChance)`) 会一直重复，直到某一次判定失败为止。

- **怪物阶级与背包生成概率** (`LOOT_DROP_RATES`):

| 怪物阶级 (Rank) | 基础生成率 (Base Chance) | 连续生成率 (Chain Chance) |
| :-------------- | :----------------------- | :------------------------ |
| 喽啰 (Minion)   | 5%                       | 50%                       |
| 精英 (Elite)    | 30%                      | 50%                       |
| 领主 (Lord)     | 30%                      | 55%                       |
| 君王一阶        | 60%                      | 60%                       |
| 君王二阶        | 65%                      | 65%                       |
| 君王三阶        | 80%                      | 80%                       |
| 远古 (Ancient)  | 90%                      | 90%                       |

- **背包内装备的境界**:
    1.  首先，系统会获取怪物所处的装备境界**阶层索引** `monsterTierInfo.tierIndex`。
    2.  然后，系统会随机选择一个目标阶层索引 `itemTierIndex`，范围是 `0` 到 `monsterTierInfo.tierIndex` (含)。
    3.  最后，在选定出的目标阶层所包含的境界范围内，随机选择一个具体的境界作为装备的境界。
    - **这意味着，一个“英雄”阶层的怪物，其背包里可能出现“英雄”、“职业者”或“佣兵”阶层的装备。**

## 3. 地图宝箱 (Grid Chests)

- **宝箱类型与概率** (`CHEST_TYPE_PROBABILITY`):

| 宝箱类型 (Type) | 出现概率 (Chance) |
| :--- | :--- |
| 木箱 (Wood) | 80% |
| 铁箱 (Iron) | 10% |
| 铜箱 (Copper) | 5% |
| 银箱 (Silver) | 3% |
| 金箱 (Gold) | 1.5% |
| 远古箱 (Ancient) | 0.5% |

- **宝箱产出 (`generateChestLoot`)**:
    - 打开宝箱可以获得**至少一件**装备，并根据宝箱类型有不同的连续掉落概率获得更多。
    - 宝箱掉落装备的品质生成，完全遵循《装备系统》中定义的**基于衰减的概率模型**。系统会以80%的概率将装备判定为该宝箱对应的**基础品质**，若判定失败，则以剩余概率的50%逐级判定更高品质，直至虚空。

- **宝箱掉落详情表**:

| 宝箱类型 | 产出装备基础品质 (Base Quality) | 连锁掉落率 (Chain Chance) |
| :--- | :--- |
| 木箱 | 精良 (Uncommon) | 50% |
| 铁箱 | 稀有 (Rare) | 55% |
| 铜箱 | 传承 (Heirloom) | 60% |
| 银箱 | 神话 (Mythic) | 65% |
| 金箱 | 传说 (Legendary) | 70% |
| 远古箱 | 史诗 (Epic) | 75% |

- **宝箱装备的境界**: 宝箱中开出装备的境界，取决于生成该批怪物时，**地图上所有怪物的最低境界索引** (`lowestMonsterRealm`)。
- **视觉反馈**: 打开宝箱时，获得的每件装备都会以其对应的**品质颜色信息**和**动态字体大小**，从宝箱地块上弹出并消失。

### 3.1 宝箱视觉效果
为了让不同等级的宝箱在地图上一目了然，每种宝箱都拥有独特的背景和边框特效：
- **木箱**: 深棕色背景，白色文字。
- **铁箱**: 青色背景，白色文字。
- **铜箱**: 橘色背景，白色文字。
- **银箱**: 银白色背景，深蓝色文字。
- **金箱**: 黄金色背景，黑色文字。
- **远古箱**: 拥有动态的、大相径庭的炫彩背景和炫彩文字，并伴有流光边框特效，极具辨识度。

## 4. 地图物品 (Grid Items)

地图上除了怪物和宝箱，还会出现两种特殊的地块：

### 4.1 体力药水
- **出现概率**: 5%
- **效果**: 拾取后立即恢复等同于玩家**当前魔防数值 (`player.stats.magicResist`)** 的体力。
- **视觉反馈**: 拾取时，一个巨大的绿色恢复数值（如 `+1234`）会从物品地块上冒出。

### 4.2 怪物镇守的宝藏 (Treasure Guarded by Monsters)
这是一个高风险、高回报的动态奖励系统，取代了原有的随机装备地块。

- **出现概率**: 每张地图上**有且仅有**一个，出现概率为5%。
- **视觉表现**:
    - 地块上显示一个巨大的、带有动态特效的问号“?”。
    - **这个“?”的颜色和特效会实时变化**，它永远代表着比“玩家在此地图上已获得的最高品质装备”**高一个等级**的品质。
    - **示例**: 地图刷新后，宝藏初始为绿色“?”（代表精良品质）。当玩家从怪物身上获得一件“稀有”装备后，地图上的“?”会立刻变为蓝色（代表传承品质），并以此类推。这为玩家清理地图提供了明确的、不断升级的视觉目标。
- **解锁条件**: **必须清除当前地图上所有的怪物和宝箱**。当满足条件后，宝藏地块会开始发光，表示可以被拾取。
- **拾取**:
    - 解锁前，该地块可被视为通路，但无法拾取。
    - 解锁后，玩家可以手动点击或通过“自动拾取”功能来获取它。
- **奖励**: 获得一件品质与“?”最终显示的品质相符的随机装备。**此奖励不会触发连锁掉落**。该系统是获取顶级品质装备的重要途径之一。

## 5. 特殊掉落 (Special Drops)

### 5.1 淬炼石 (Tempering Stone)

- **简介**: 一种用于强化装备词条的特殊消耗品。
- **掉落机制**:
  - 淬炼石的掉落判定与装备掉落**完全独立**，互不影响。一个怪物可能同时掉落装备和淬炼石。
  - **掉落概率公式**: `怪物神性 * 0.5%`。
    - 对于非“远古”阶级的敌人，此掉落率**最高不超过10%**。
  - **示例**: 一个拥有300点神性的“君王”阶级怪物，其理论掉落率为 `300 * 0.005 = 150%`。由于阶级限制，其实际掉落率会被强制修正为**10%**。而一个拥有300点神性的“远古”阶级怪物，其掉落率为150%，意味着它**必定掉落1颗**淬炼石，并有**50%的额外几率掉落第2颗**。
