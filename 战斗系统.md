# 战斗系统设计文档 (Combat System Design)

战斗是游戏的核心循环，其设计经过重构，移除了中断性的警告，并引入了更具互动性的回合制交锋，旨在提供更流畅和更具策略性的体验。

## 1. 战斗流程 (Combat Flow)

战斗结果的预判仍然在 `calculateCombatOutcome` 函数中进行模拟，这是一个纯粹的全自动回合制计算过程，用于判断最终的胜负。

1.  **初始化**:
    - 玩家体力 (`pHealth`) 和怪物体力 (`mHealth`) 被初始化。
    - `totalRawDamageFromMonster` (怪物造成的原始伤害累计) 初始化为 0。
    - 回合数 `rounds` 从 1 开始。
    - **伤害预计算**: `playerDamage = max(0, playerStats.attack - monster.stats.defense)` 和 `monsterDamage = max(0, monster.stats.attack - player.stats.defense)`。

2.  **回合循环 (Loop)**: 循环会持续进行，直到一方体力耗尽。**没有固定的回合上限**，允许出现极高的回合数。

    a. **玩家回合**:
        - **敌人扣血**: `mHealth -= playerDamage`
        - **玩家回血**: `pHealth += playerStats.lifeRegen`
        - **胜利判定**: 如果 `mHealth <= 0`，循环结束，玩家胜利。

    b. **敌人回合**:
        - **玩家扣血**: `pHealth -= monsterDamage`
        - **累计原始伤害**: `totalRawDamageFromMonster += monsterDamage`
        - **失败判定**: 如果 `pHealth <= 0`，循环结束，玩家失败。

3.  **战斗结束条件**:
    - **一方胜利**: 如上所述，某一方的体力值小于或等于 0。
    - **僵局 (Stalemate)**: 在战斗开始前会进行一次预判。如果玩家的每回合伤害 `playerDamage` <= 0，则判定玩家无法取胜，战斗直接被视为失败。此情况 `rounds` 会被记为 `Infinity`。

## 2. 胜负结算 (Combat Outcome)

战斗模拟的结果 (`CombatOutcome`) 会被用于界面预显示和实际战斗结算。

### 2.1. 玩家胜利 (Player Won)

当玩家点击一个预判会胜利的敌人时，会直接触发完整的胜利结算。

- **体力结算 (Health Change)**:
    1.  计算**最终承受伤害**: 战斗中累计的 `totalRawDamageFromMonster` 在此时才统一被玩家的魔防减免。
        `finalDamageTaken = max(0, totalRawDamageFromMonster - playerStats.magicResist)`
    2.  计算**总体力恢复**:
        `totalRegen = playerStats.lifeRegen * rounds`
    3.  计算**最终体力变化**:
        `healthChange = totalRegen - finalDamageTaken`
    - 这个结果可能是正数（恢复体力）、负数（损失体力）或零。

- **奖励 (Rewards)**:
    - **修为获取**: 只有当 `monster.realmIndex >= player.realmIndex` 时，才能获得 `monster.xpReward`。
    - **战利品**: 有几率获得装备，详见 `战利品系统.md`。

### 2.2. 手动回合制交锋 (Manual Round-based Exchange)

为了提供更流畅和更具策略性的体验，当玩家手动点击一个预判会失败的敌人时，系统将不再弹出任何警告，而是进入一个即时的回合制交锋模式。

- **触发条件**: 玩家手动点击一个 `combatOutcome.playerWon` 为 `false` 的怪物。
- **交锋机制**:
    1.  点击后，**玩家与怪物会立即进行一回合的攻防交换**。
    2.  **视觉反馈**: 双方会同时向对方发射攻击弹道，并在各自的地块上弹出该回合所受的伤害数字。
    3.  **持续交锋**: 玩家可以**重复点击**该怪物，每一次点击都会触发新一回合的攻防交换。
    4.  **最终结算**: 这个过程会一直持续，直到某一方的体力小于等于0。届时，系统会根据最终的胜负结果，触发完整的胜利或失败结算逻辑。

### 2.3. 被击溃的后果 (Consequences of Defeat)

当玩家在战斗中被击败（无论是因为自动战斗，还是被“敌人主动进攻系统”击溃，或是在手动回合交锋中落败），会触发以下一系列后果：
1.  **弹窗提示**: 出现一个“你被击溃了！”的模态框。
2.  **失去资源**: 玩家当前拥有的所有 `淬炼石` 和 `金币` 都会被清零。
3.  **体力恢复**: 玩家会恢复一定的体力，数值等于 `玩家攻击力 + 玩家防御力 + 玩家魔防`。
4.  **地图重置**: 当前的整个战斗网格会被刷新。
5.  **挑战失败**: 如果玩家正处于任何“境界挑战”或“楼层挑战”中，该挑战会被强制终止。

## 3. 战斗视觉反馈 (Combat Visual Feedback)

为了增强战斗的打击感和奖励的满足感，当击败敌人或拾取物品时，会在其地块上触发一系列弹出式动画。

- **反馈类型**:
  - **伤害/治疗**:
    - 如果玩家在战斗中**损失体力**，一个巨大的红色伤害数字（如 `-1234`）会从怪物地块上冒出。
    - 如果玩家在战斗中**恢复体力**，一个巨大的绿色恢复数字（如 `+5678`）会冒出。
    - 如果是0伤，则不显示任何数字。
  - **获取经验**: 如果玩家获得了修为奖励，一个巨大的紫色经验数值（如 `+100 XP`）会冒出。
  - **获取装备**:
    - 每获得一件装备，一条带有该装备**品质颜色**的简化信息（如 `获得了 [传说]装备`）会从对应的怪物或宝箱地块上弹出并消失。
    - 如果获得淬炼石，则会冒出 `获得了淬炼石*1` 的信息。

- **动画效果与序列**: 所有反馈信息都使用相同的“冒出并消失”的抛物线动画。为了获得更清晰、更具节奏感的视觉效果，当一次事件产生多个反馈时，它们将以一个快速的**序列**依次弹出，而不是同时出现，在视觉上形成一种连贯的“爆裂”效果。

- **字体大小与品质挂钩**: 掉落装备信息的弹出文字，其**字体大小**会根据装备品质动态调整。品质越高，字体越大，为玩家获得高级战利品时提供更强烈的视觉冲击和满足感。
