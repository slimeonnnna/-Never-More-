# 境界系统设计文档 (Realm System Design)

境界是玩家成长的核心体现，直接影响玩家的基础属性和升级节奏。

## 2. 属性成长 (Attribute Growth)

- **初始属性**: 玩家创建时，会获得第一个境界（下级佣兵）的属性加成（50生命，20属性点）。这20点属性点会通过 `Math.floor(Math.random() * (20 + 1))` 随机分配给攻击力，余下的点数则全部给予防御力。同时，玩家会根据初始属性点获得额外的魔防加成，数值为 `20 * 1.5 = 30`。
- **升级/突破属性**: 当玩家境界提升时，会获得新旧境界之间的属性差值。新增的属性点将通过以下**确定性**逻辑分配：
  - `新增攻击力 = Math.floor(新增属性点 / 2)`
  - `新增防御力 = 新增属性点 - 新增攻击力`
  - `新增魔防 = Math.round(新增属性点 * 1.5)`
  - 所有新增属性都会直接累加到角色的永久加成 (`permanentBonuses`) 中。

## 3. 升级机制 (Leveling Mechanism)

- **获取修为 (XP)**: 玩家通过击败任何怪物都能获得修为。修为的奖励由两部分精确组成：
    1.  **基础修为**: 由怪物的境界等级决定，计算公式为 `2 ^ 怪物境界索引`。例如，`下级佣兵` (索引0) 提供 `2^0 = 1` 点基础修为，`中级佣兵` (索引1) 提供 `2^1 = 2` 点基础修为，以此类推。
    2.  **阶级倍率**: 基础修为会乘以怪物阶级（如喽啰、精英、领主等）对应的经验倍率（`xpMultiplier`）。例如，一个“精英”级（5倍）的“高级佣兵”（基础修为4），将提供 `4 * 5 = 20` 点总修为。
- **自动升级**: 在绝大部分境界，当 `当前修为 >= 下一级所需修为` 时，系统会自动扣除所需修为，并将玩家提升至下一个境界，同时获得属性成长。
- **手动突破 (Breakthrough)**:
    - **突破点**: 在特定的瓶颈境界（见上表**加粗**项），自动升级会暂停。这些境界的索引被精确定义在 `constants.ts` 的 `BREAKTHROUGH_REALM_INDICES` 数组中: `[6, 13, 24, 31, 34, 37, 40, 45, 55, 60]`。
    - **突破条件**: 玩家必须积累足够的修为 (`xp >= xpToNextLevel`)，然后手动点击角色面板的“突破”按钮。
    - **突破效果**: 成功突破后，玩家进入下一个大的阶段，并自动重置“搜寻限制”中的“战力阶级限制”为“无限制”。
