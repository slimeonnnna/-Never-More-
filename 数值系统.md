# 数值系统设计文档 (Number System Design)

本文档旨在阐述游戏中大数值的显示规则，该系统确保了游戏后期庞大的数值能以清晰、易于理解的中文单位进行呈现。

## 1. 核心格式化规则 (Core Formatting Rules)

游戏中的所有数值显示都遵循 `formatLargeNumber` 函数中定义的逻辑：

- **常规数值**: 任何小于 `1,000,000` (一百万) 的数字，将以标准的千位分隔符格式显示（例如：`999,999`）。
- **大数值**: 当数字达到或超过 `1,000,000` 时，将启用自定义的中文单位系统进行格式化。
- **取整规则**: 所有经过单位格式化的数值都会向下取整，小数部分会被舍弃。例如，`1.23万` 将会显示为 `1万`。
- **特殊值**: `Infinity` 会被显示为 `MAX`。

## 3. 显示逻辑详解 (Detailed Display Logic)

系统在选择使用哪个单位时，并非简单地使用最大的适用单位，而是遵循一个特殊的“延迟晋级”规则以优化可读性。

### 规则 1: 默认使用最大单位
在一般情况下，系统会寻找能包含该数字的最大单位。
- **示例**: 数字 `1,234,567,890,123`
  - 它大于 `1兆` (`10^12`) 但小于 `1京` (`10^16`)。
  - 因此，系统默认会使用“兆”作为单位。
  - 计算结果为 `floor(1,234,567,890,123 / 10^12)`，最终显示为 `1兆`。

### 规则 2: “十万”延迟晋级规则
这是系统显示逻辑的核心。在确定最大单位后，系统会“回顾”一下次级单位。

- **逻辑**: 如果一个数字用**次一级**的单位来表示时，其整数部分**小于 `100,000`**，那么系统将**优先使用这个次级单位**，而不是晋级到更大的单位。

- **正面案例 (触发规则)**: 数字 `999,990,000`
  1. 系统识别出最大可用单位是“亿”。
  2. 系统回顾次级单位“万”。
  3. 用“万”来表示该数字，结果是 `99999万`。
  4. 因为 `99999` 小于 `100,000`，满足延迟晋级规则。
  5. **最终显示**: `99999万` (而不是 `9亿`)。

- **反面案例 (不触发规则)**: 数字 `1,000,000,000`
  1. 系统识别出最大可用单位是“亿”。
  2. 系统回顾次级单位“万”。
  3. 用“万”来表示该数字，结果是 `100000万`。
  4. 因为 `100000` **不小于** `100,000`，不满足延迟晋级规则。
  5. **最终显示**: `10亿` (使用原定的最大单位“亿”)。
