# 楼层系统设计文档 (Floor System Design)

楼层系统是游戏后期内容的核心，为玩家提供了不断挑战自我、追求更高品质战利品的目标。它通过一个清晰的界面和一套动态的概率调整机制，极大地丰富了游戏的策略深度和可玩性。

## 1. 楼层控制面板

在战斗区域下方，有一个专用的楼层控制面板，包含了玩家进行楼层管理所需的所有功能。

- **`当前楼层`**: 静态显示玩家目前所在的楼层，让玩家对当前所处环境一目了然。
- **`上一层`**: 点击后，如果当前楼层大于1，玩家会返回上一层，并刷新地图。在第1层时此按钮会被禁用。
- **`下一层 / 挑战更高层`**: 这是一个动态按钮，其行为取决于玩家的进度：
  - **情况一 (已解锁)**: 如果当前楼层**低于**玩家已达到的最高楼层，此按钮显示为“下一层”。点击后，玩家会直接前进到下一层，并刷新地图。
  - **情况二 (未解锁)**: 如果当前楼层**等于**玩家已达到的最高楼层，此按钮变为“挑战更高层”。点击后将开启楼层晋升挑战，是玩家探索未知领域的唯一途径。
- **`快速前往`**: 点击后弹出一个窗口，允许玩家输入一个已解锁的楼层号（1 到最高楼层之间），确认后直接跳转到该楼层并刷新地图。
- **`上楼`**: 当玩家成功完成楼层挑战后，整个楼层控制面板会变成一个醒目的“上楼”按钮。点击后，玩家才会正式晋升到新的楼层。

在任何挑战（包括境界挑战和楼层挑战）进行中，上述所有按钮都会被禁用，以确保玩家能专注于当前挑战。

## 2. 楼层效果：动态概率调整

当前所在的楼层 (`currentFloor`) 会极大地影响怪物和装备的生成概率，其核心规则是**“楼层越高，下限越高”**。系统会动态调整最低级物品的出现概率，并将这些概率重新分配给更高级的物品。

### 2.1 装备品质影响 (`generateEquipment`)

- **概率公式**: `修正后的普通品质概率 = Math.max(0, 0.80 - ((currentFloor - 1) * 0.01))`
- **机制**: 从第2层开始，每提升一层，“普通(Common)”品质装备的基础出现概率 (80%) 就会降低1%。**第1层没有此效果**。
- **示例**:
  - 在第1层，普通概率为 `80% - (0 * 1%) = 80%`。
  - 在第2层，普通概率为 `80% - (1 * 1%) = 79%`。
  - 在第81层，普通概率为 `80% - (80 * 1%) = 0%`。
- **概率再分配**: 减少的低级概率会被系统自动、按比例地分配给更高等级的品质，使得在高楼层中获得高级战利品变得更加普遍。

### 2.2 怪物境界影响 (`_createMonster`)

- **概率公式**: `修正后的最低境界概率 = Math.max(0, 0.80 - ((currentFloor - 1) * 0.01))`
- **机制**: 从第2层开始，每提升一层，最低级境界“下级佣兵”的基础出现概率 (80%) 就会降低1%。**第1层没有此效果**。
- **示例**:
  - 在第1层，下级佣兵概率为 `80% - (0 * 1%) = 80%`。
  - 在第2层，下级佣兵概率为 `79%`。
  - 在第81层，下级佣兵概率为 `0%`。
- **概率再分配**: 同样，减少的概率会被重新分配给更高境界的怪物。

## 3. 楼层晋升挑战

- **目的**: 解锁并前往一个全新的、更高的楼层。
- **触发**: 在玩家的最高楼层点击“挑战更高层”按钮。
- **挑战内容**: 系统会生成一个特殊的挑战地图，该地图上仅有**一个“远古”阶级**的怪物。该怪物的境界被精确锁定为**玩家当前楼层所能生成的最低境界**。
  - **核心逻辑 (`getLowestPossibleRealmIndex`)**: 系统会根据当前楼层的数值计算出届时概率不为0的最低境界。
  - **示例**: 玩家在第80层，挑战第81层。系统计算得出，在第80层时，“下级佣兵”的出现概率已降为0，新的最低境界变为“中级佣兵”。因此，挑战内容就是击败一个“远古级”的“中级佣兵”。
- **胜利条件**: 玩家只需**击败地图上的那个怪物**即可完成挑战。
- **胜利效果**: 挑战成功后，楼层控制面板会变成一个醒目的“上楼”按钮，日志会弹出祝贺信息。
- **晋升**: 点击“上楼”按钮后，玩家的“最高楼层”和“当前楼层”记录会+1，并自动前进到这个新的楼层，同时刷新地图以展示新楼层的内容。
