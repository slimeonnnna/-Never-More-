# 装备系统设计文档 (Equipment System Design)

装备是提升玩家战力的主要途径，其设计围绕品质、境界、词条和战力四个核心维度。

## 1. 装备品质 (Quality)

装备根据稀有度分为不同品质，品质越高，生成时获得的随机词条（属性）数量越多。

| 品质等级 | 中文名 | 英文名 | 词条数量 | 文本样式 (CSS Class) |
|:---:|:---:|:---:|:---:|:---|
| 1 | 普通 | Common | 1 | `text-gray-400` |
| 2 | 精良 | Uncommon | 2 | `text-green-500` |
| 3 | 稀有 | Rare | 3 | `text-blue-500` |
| 4 | 传承 | Heirloom | 4 | `text-indigo-400` |
| 5 | 神话 | Mythic | 5 | `text-purple-500` |
| 6 | 传说 | Legendary | 6 | `text-legendary` (橙色渐变) |
| 7 | 史诗 | Epic | 7 | `text-epic` (品红渐变) |
| 8 | 亘古 | Ancient | 8 | `text-ancient` (玫瑰红渐变) |
| 9 | 至尊 | Supreme | 9 | `text-supreme` (青柠色渐变) |
| 10 | 混沌 | Chaos | 10 | `text-chaos` (彩虹动态渐变) |
| 11 | 虚空 | Void | 11 | `text-void` (白色动态渐变) |

- **超·虚空品质与进化**: "虚空"品质之上，存在"虚空N重"的进阶等级，并拥有独特的进化机制。
  - **生成**: 当一次品质判定轮空所有11个基础品质后，会进入虚空品质的判定。它首先被设定为 `虚空1重` (12词条)，之后有极小概率连续晋升为 `虚空2重`, `虚空3重`...
  - **属性**: 新生成的虚空装备**必定附带至少一条“神性”词条**。
  - **进化 (吞噬)**: 虚空装备可以通过吞噬同部位的其他虚空装备来进化。
    - **触发条件**: 当玩家通过自动拾取或战斗获得一件新的虚空装备，且其对应装备槽上已装备了另一件虚空装备时，将自动触发吞噬。
    - **胜负规则**: 严格遵循以下优先级：
      1.  **高境界**: 装备的境界阶层 (`佣兵`, `职业者`等) 更高者胜出。
      2.  **高神性**: 在同一境界阶层下，拥有更高“神性”词条的虚空装备会成为胜利者。
      3.  **高战力**: 在境界阶层和神性都相同的情况下，战力更高者胜出。
    - **进化效果**:
      1.  **重数提升**: 胜利者的虚空重数会提升。新的重数等级为 `胜利者重数 + 失败者重数 + 1`。例如，一件`虚空2重` (等级2) 的武器吞噬了一件`虚空1重` (等级1) 的武器，它将进化为 `2 + 1 + 1 = 4`，即`虚空4重`。一件`虚空` (等级0) 吞噬另一件`虚空` (等级0)，将进化为 `0 + 0 + 1 = 1`，即 `虚空1重`。
      2.  **词条增加**: 每次成功吞噬后，胜利装备会**获得新的随机词条**。新增词条的数量等于 `被吞噬装备的虚空重数 + 1`。这些新词条的生成规则与普通装备生成词条的规则一致。

## 2. 品质生成概率 (Quality Generation Probability)

装备的品质由一个精确的、基于衰减的概率模型决定 (`generateEquipment`)，并且受到**楼层系统**的显著影响。

1.  **基础概率与楼层影响**:
    - **基础概率**: 在不考虑楼层的情况下，怪物掉落的装备有 **80%** 的基础概率成为“普通(Common)”品质。
    - **楼层修正**: 玩家所在的**当前楼层 (`currentFloor`)** 会直接降低这个基础概率。
      - `修正后的基础概率 = Math.max(0, 0.80 - ((currentFloor - 1) * 0.01))`
      - 这意味着从**第2层开始**，每提升一层，“普通”品质的出现概率就降低1%，第1层没有此效果。例如，在第41层，普通品质的概率为 `80% - ((41-1)*1%) = 40%`。在第81层及以上，这个概率会降至0。

2.  **衰减概率**: 若未能成为“普通”品质，则进入更高品质的判定。下一级品质的出现概率是 **当前剩余总概率的50%**。
    - **示例 (在第41层，普通概率为40%)**:
        - **精良(Uncommon)**: `(1 - 0.4) * 0.5 = 30%`
        - **稀有(Rare)**: `(1 - 0.4 - 0.3) * 0.5 = 15%`
        - **传承(Heirloom)**: `(1 - 0.4 - 0.3 - 0.15) * 0.5 = 7.5%`
    - ...这个过程以此类推，贯穿所有品质等级。最后剩余的极小概率将用于生成上文提到的“超·虚空”品质。
    - **宝箱掉落**: 对于宝箱掉落，如果其基础品质不是“普通”（例如金箱的基础品质是“传说”），则不受楼层修正影响，直接从其设定的基础品质开始进行品质判定。

## 3. 装备境界 (Equipment Tiers)

装备的属性数值与“装备境界”强相关，这决定了其属性的量级。

- **境界阶层表**: `EQUIPMENT_REALM_TIERS`

| 阶层名称 | 包含境界索引 (Realm Index Range) | 阶层索引 (tierIndex) |
|:---:|:---:|:---:|
| 佣兵 | 0 - 6 | 0 |
| 职业者 | 7 - 13 | 1 |
| 英雄 | 14 - 24 | 2 |
| ... | ... | ... |

- **数值膨胀**: 每提升一个装备境界阶层（`tierIndex`），其主属性（攻击、防御、魔防）的 **最大值** 会乘以 `50` 的 `tierIndex` 次方 (`Math.pow(50, itemTierInfo.tierIndex)`)。
    - **核心规则 (`generateEquipment` a)**: 对于 `职业者` 及以上阶层的装备 (`tierIndex > 0`)，其属性的**最小值被硬编码为1** (`min = 1`)。
    - **核心规则 (`generateEquipment` b)**: 最终生成的属性值还会受到一个微调加成：`finalValue = Math.round(value * (1 + (itemRealmIndex / 200)))`。这意味着同一阶层内，更高境界的装备会略强一些。

## 4. 装备词条 (Attributes)

- **词条权重与基础数值表** (来源: `ATTRIBUTE_GENERATION_CONFIG`):

| 词条类型 | 生成权重 (Weight) | 基础数值范围 (佣兵阶层) |
|:---:|:---:|:---:|
| `攻击力` | 300 | [1, 10] |
| `防御力` | 300 | [1, 10] |
| `魔防` | 300 | [1, 30] |
| `体力恢复` | 75 | [1, 6] |
| `神性` | 25 | [1, 2] |

- **特殊词条规则**:
    - **神性**: 该属性 **不受** 50倍的境界膨胀加成。其随机最大值随装备境界**阶层索引 (`tierIndex`)** 的提升而+1。公式为 `max = baseMax + itemTierInfo.tierIndex`。
    - **体力恢复**: 享受50倍的境界膨胀加成，但只提升随机范围的**最大值**，最小值不受影响。

## 5. 装备战力 (Combat Power)

战力是衡量装备综合强度的一个计算值，用于快速比较装备优劣。

- **计算公式** (来源: `calculateItemCombatPower`):
  `战力 = (攻击力 * 2) + (防御力 * 2) + (魔防 * 1) + (体力恢复 * 10) + 战力修正(神性)`

- **神性的战力修正**:
    - 如果装备**同时拥有** `神性` 和 `攻击力` 词条，则修正值为: `神性 * 攻击力`。
    - 如果装备**只有** `神性` 词条，系统会通过以下公式估算一个平均攻击力用于计算：
        1.  获取该装备境界阶层下，攻击力词条的基础最大值 `baseMax` (即10)。
        2.  计算该阶层的理论最大攻击力 `maxAttack = baseMax * Math.pow(50, itemTierInfo.tierIndex)`。
        3.  `估算的平均攻击力 = Math.floor((1 + maxAttack) / 2)`。
        4.  最终修正值为: `神性 * 估算的平均攻击力`。

## 6. 属性效果 (Attribute Effects)

部分属性除了提供基础数值外，还具有特殊的加成效果。

- **神性 (Divinity) 的复利增强**:
  - “神性”是后期成长最重要的属性之一，它能以**复利**形式显著增强角色的其他属性。
  - **攻击力与防御力**: 每拥有 **1** 点神性，角色的最终攻击力和防御力就会乘以 **1.01**。
    - 例如，一个拥有3点神性的角色，其最终攻击力将是 `基础攻击力 * 1.01 * 1.01 * 1.01`。
  - **魔防**: 每拥有 **1** 点神性，角色的最终魔防就会乘以 **1.03**。

## 7. 装备词条完美度 (Attribute Perfection)

为了给玩家提供更精细的追求目标，每条装备词条的右侧会显示一个百分比，代表该词条数值的“完美度”。

- **显示位置**: 装备的浮窗提示（Tooltip）和详情模态框（Modal）。
- **核心作用**: 此百分比仅为直观展示词条的随机结果优劣，**不以任何方式影响装备的实际战力计算**。

### 计算逻辑

完美度的计算基于词条的当前值与其在该装备上可能达到的理论最大值之比。

1.  **单次生成的词条**:
    - 完美度 = `(实际随机值 / 该词条在当前装备境界下的理论最大值) * 100%`
    - **示例**: "佣兵"阶层的"攻击力"词条，理论范围是[1, 10]。如果随机到 `5`，则完美度为 `(5 / 10) * 100% = 50%`。

2.  **多次生成的同一词条**:
    - 当一件高品质装备多次生成了同一类型的词条时（例如，一件“史诗”装备roll了3次“攻击力”），其理论最大值会累加。
    - 理论总最大值 = `单次roll的理论最大值 * 生成次数`
    - 完美度 = `(词条最终总值 / 理论总最大值) * 100%`
    - **示例**: 一件"职业者"装备3次roll到"攻击力"，该阶层单次roll理论最大值为`500`。那么理论总最大值是`1500`。如果3次roll的总和是`900`，则完美度为 `(900 / 1500) * 100% = 60%`。

此设计旨在为玩家在刷装备时提供一个除了战力以外的、更细致的毕业追求，增加了游戏的深度和可玩性。

## 8. 装备强化：淬炼 (Equipment Enhancement: Tempering)

淬炼是提升现有装备词条属性的唯一方式，为玩家提供了毕业装备的终极追求。

- **核心物品：淬炼石**
  - **来源**: 击败怪物时有几率掉落，详见《战利品系统》。
  - **特性**: 一种可叠加的消耗品，不会自动使用。在UI上会以“传说”品质的视觉效果来表示，但它本身没有词条，不属于装备。

- **使用方法**
  - 在装备详情界面，任何一个“完美度”**未达到100%** 的词条旁边，都会出现一个“淬炼”按钮。
  - 点击按钮会消耗一颗淬炼石并对该词条进行强化。完成之后，装备详情窗口会自动关闭，同时在游戏日志中会显示详细的属性变化结果，整个过程不会有额外的弹窗确认。

- **强化效果**
  - **基础效果**: 每次淬炼会提升词条 **1%** 的完美度。
  - **保底机制**: 为了确保每次强化都有实质性提升，系统设置了保底规则：
    1.  系统首先计算提升1%完美度后，词条的数值是否会增加（取整后）。
    2.  **情况一 (数值无变化)**: 如果增加1%完美度后，词条的整数值没有变化（例如，从50%变为51%，但数值依然是5），则系统会强制为该词条的数值 **+1**。此时，该词条的完美度会根据新的数值被重新计算。
    3.  **情况二 (数值有变化)**: 如果增加1%完美度后，词条的整数值**能够至少+1**，则系统会正常应用这个百分比换算后的新数值，不触发保底。

- **示例**:
  - 玩家有一件“佣兵”阶层的装备，上面有一条词条“攻击力+5”。该阶层攻击力理论最大值为10，因此这条词条的完美度是 **50%**。
  - 玩家使用淬炼石。系统计算1%完美度的提升，发现数值依然是5。
  - 触发保底机制，词条数值变为 `5 + 1 = 6`。
  - 系统根据新数值6重新计算完美度，` (6 / 10) * 100% = 60% `。
  - **最终结果**: 词条变为“攻击力+6”，完美度变为60%。

## 9. 装备命名系统 (Equipment Naming System)

为了增加装备的多样性和独特性，游戏采用了一套程序化的随机命名系统，取代了固定的名称库。每件装备的名称都由“前缀”和“基础物品名”动态组合而成。

### 9.1 命名规则

装备名称的生成遵循以下规则：

1.  **随机前缀**:
    - 系统会从一个包含100个充满意象的汉字词库中，随机选择 **1到4个字** 作为前缀。
    - 例如，可能会生成“云”、“风月”或“飘渺玄梦”等前缀。
2.  **“之”字连接词**:
    - 在前缀生成后，有 **10%** 的概率在前缀和基础物品名之间插入一个“之”字，增加名称的古典韵味。
    - 例如，“风月”前缀可能会变为“风月之”。
3.  **基础物品名**:
    - 系统会根据装备的**部位 (`slot`)**，从一个对应的基础物品名词库中随机选择一个作为后缀。
    - 例如，如果装备是“武器”，可能会选择“剑”、“刀”或“斧”等基础名。

### 9.2 名称示例

- `[佣兵][传说] 飘云剑`
- `[英雄][史诗] 风月之冠`
- `[主宰][混沌] 墨影星辰袍`

这个系统确保了玩家在游戏中获得的每一件装备都拥有一个独特的、富有诗意的名字，极大地增强了探索和收集的乐趣。

## 10. 炼化与奇迹 (Refining & Miracles)

炼化是处理多余装备、获取资源并追求意外惊喜的核心机制。

- **标准炼化**:
  - **效果**: 玩家可以将任何装备炼化，立即获得等同于该装备 **战力值 (`item.combatPower`)** 的金币。
  - **目的**: 这是游戏中最主要的金币来源，也是清理背包、处理低价值装备的有效手段。

- **炼化奇迹**:
  - **触发**: 在炼化任何一件非顶级品质的装备时，有 **5%** 的固定概率触发“炼化奇迹”。
  - **效果**: 触发后，系统**不会**给予金币，而是会生成一件全新的、品质**提升一个等级**的装备。
    - **完全重置**: 这件新装备的**部位 (`slot`) 和所有词条 (`attributes`) 都会被完全重新随机生成**，其属性数值将符合新的、更高的品质和原装备的境界等级。
    - **神性保障**: 如果新生成的装备品质达到了“传说”或更高，系统会确保它**必定带有一条“神性”词条**，保证了高品质装备的核心价值。
  - **示例**: 炼化一件“史诗”品质的武器时触发奇迹，玩家可能会获得一件全新的“亘古”品质的头盔、裤子或任何其他部位的装备。
