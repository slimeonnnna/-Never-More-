# 友军系统设计文档 (Friendly Unit System Design)

友军系统为游戏世界增添了新的维度，玩家有机会在探索中遭遇强大的、非敌对的NPC。这些友军不仅是玩家实力的参照物，更通过其身上的神兵利器，直观地向玩家展示了高级装备的强大威力，从而提供了更明确的追求目标。

## 1. 友军的生成与识别

- **出现时机**: 在每次“搜寻”刷新地图后，系统会判定当地图上存在怪物时，将**战力最高**的那个怪物进行一次检定。
- **出现概率**: 该检定有 **10%** 的概率成功，使这个最强的怪物转变为“友军单位”。
- **豁免情况**: 为了保证挑战的纯粹性，在“楼层挑战”期间 **不会** 生成友军。
- **视觉标识**:
    - 转变后的友军地块，其右上角会显示一个醒目的**粉色“友”字**，清晰地将其与普通敌人区分开。
    - 该地块的类型会从 `monster` 变为 `friendly`，使其在系统层面被识别为非敌对单位。
- **日志提示**: 当友军出现时，游戏日志会显示一条特殊信息：“你似乎遇到了一个友善的强者...”，增添了不期而遇的惊喜感。

## 2. 友军的行为与交互

- **基本行为**: 友军是和平的，**不会主动攻击**。其地块被视为**可通行区域**，玩家的移动路径可以自由穿过。
- **实力参照**: 友军地块上依然会像普通怪物一样，显示玩家与其进行理论战斗的**预估体力损耗**。这为玩家提供了一个直观的、无风险的方式来衡量自己与这位强者的实力差距。
- **交互选项**: 手动点击友军地块会弹出一个交互菜单，提供两个选项：
  1. **查看**: 打开“友方详情”模态框，可以洞悉其属性、装备和背包。
  2. **战斗**: 玩家可以主动选择与友军进行战斗。战斗规则与攻击普通怪物完全一致。

## 3. 友军详情面板。

这是一个只读的信息展示窗口，旨在让玩家“洞悉强者”的秘密。

- **装备栏**: 完整展示友军当前装备的十二件物品，布局与玩家的角色装备栏一致。
- **背包栏**: 展示该友军“携带”的随机物品。这些物品是根据该友军的实力，通过标准的战利品生成逻辑 (`generateLoot`) 模拟掉落而产生的。**此过程不保证背包中一定有物品，可能为空**。
- **只读交互**: 玩家可以点击友军的任何一件装备或背包物品，来打开标准的“装备详情”窗口查看其具体词条和属性。但是，所有的操作按钮（如装备、替换、炼化、淬炼）都会被**隐藏**，确保玩家只能查看而无法进行任何操作。

## 4. 核心机制：装备属性反推工程

为了确保友军的强大“有据可查”且“真实可实现”，其装备生成采用了一套精确的**“属性反推”**机制。

- **设计目标**: 友军展示的装备，必须能**精确对应**其在图鉴上显示的面板属性。如果一名玩家集齐了这套装备，其最终面板属性将与友军的数值完全一致。

- **反推逻辑 (`generateCharacterSheet`)**:
  1.  **锁定目标属性**: 系统首先将友军（原怪物）的面板属性（攻击、防御、神性）作为最终要达成的目标。
  2.  **计算玩家基础值**: 系统会精确计算一个与友军同境界的玩家角色所拥有的基础属性（主要来源于境界提升的永久加成）。
  3.  **计算装备需求**: 通过 `友军总属性 - 玩家基础属性` 的公式，系统可以反向计算出需要由**一整套12件装备**共同提供的属性净值。这个计算过程会严谨地考虑“神性”对攻防的百分比加成效果，从而保证结果的精确性。
  4.  **生成单件装备**: 系统会逐一为12个装备槽生成装备。
      a. **核心属性分配**: 首先，系统通过随机算法，将计算出的总核心属性需求（总攻击、总防御、总神性）精确地分配到12个装备槽位上。
      b. **附加“展示性”词条**: 其次，系统会**有几率**为每个装备槽额外生成对玩家有用的“魔防”和“体力恢复”词条。这些词条的数值会严格按照玩家获取装备时的规则（考虑装备境界）进行随机生成，确保其合理性。这些词条仅供玩家参考，**不会**影响友军自身的面板属性。
      c. **计算等效词条数**: 系统会估算要达到该装备槽所需的属性值，需要多少次标准的“词条生成”（Roll）。
        -  首先，系统会计算出在该境界下，每种属性（攻击、神性等）**单次生成**所能达到的理论最大值。
        -  然后，用该装备槽需要的属性值除以这个“单次最大值”，向上取整，就得出了该属性所需的“等效词条数”。例如，如果一件装备需要提供1000点攻击力，而该境界下攻击力单次生成的最大值是250，那么就需要 `ceil(1000/250) = 4` 个等效词条。
        -  系统会为装备槽上的所有属性（攻击、防御、神性等）计算等效词条数，并将其**总和**作为该装备的总词条数。
      d. **精确匹配品质**: **装备的最终品质将严格由这个计算出的“总词条数”决定**。
        -  例如，如果一件装备经过计算总共需要7个等效词条，那么它的品质就**必定是“史诗”**（7词条）。
        -  如果总词条数超过11个，它将被正确地判定为“虚空N重”品质，例如，15个等效词条会对应生成一件“虚空5重”的装备。
        -  装备上每条属性的“生成次数”(`rollCount`)也会被精确记录，确保其“完美度”百分比的计算准确无误。

- **最终效果**: 这一机制确保了友军展示的每一件装备都是“真实”的。玩家看到的不再是一个遥不可及的战力数值，而是可以通过检查其装备，清晰地理解“原来在这个境界，凑齐这样一套装备，就能达到这种实力”。这为玩家提供了具象化的、可验证的、可追赶的毕业目标，极大地增强了刷装备的动力和乐趣。
